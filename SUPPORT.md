# Support

Thanks for using Receiptd! Here's where to go depending on what you need.

## Usage questions

For "how do I..." questions, configuration help, or general discussion,
please use [GitHub Discussions](https://github.com/harveysandiego/receiptd/discussions)
rather than opening an issue. This keeps the issue tracker focused on
actionable bugs and feature work, and lets other users find answers to the
same question later.

## Bug reports

If you've found something that doesn't work as documented, please open an
issue using the **Bug report** template. Include your Receiptd version,
deployment method (Docker / Raspberry Pi / bare metal), printer model if
relevant, and enough detail to reproduce the problem — see
[CONTRIBUTING.md](CONTRIBUTING.md) for what a good bug report looks like.

## Feature requests

Open an issue using the **Feature request** template. If your idea touches
the core Receipt model, REST API, or package layout, please skim
[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) first — it may already be on
the roadmap, or there may be a documented reason the design looks the way
it does.

## Security issues

Do not use issues or discussions for security vulnerabilities — see
[SECURITY.md](SECURITY.md) for private reporting instructions.

## Contributing

If you'd like to fix a bug or build a feature yourself, see
[CONTRIBUTING.md](CONTRIBUTING.md) for development setup, coding standards,
and how to propose changes.

## Response times

Receiptd is maintained as an open-source project outside of a full-time
job. There's no guaranteed response time, but issues and PRs are reviewed
regularly.
